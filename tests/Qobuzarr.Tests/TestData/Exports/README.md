# Lidarr Export Test Data

This directory contains test data files generated by the Lidarr export functionality for unit testing.

## Files

### `lidarr-wanted-albums-sample.json`
- **Format**: JSON
- **Albums**: 25 classic albums from the 1960s-1990s
- **Purpose**: Comprehensive test data with full metadata
- **Features**: Includes all export fields with realistic data

### `lidarr-test-albums.json`
- **Format**: JSON  
- **Albums**: 10 modern albums (2014-2023)
- **Purpose**: Focused test data for specific test scenarios
- **Features**: Mix of album types, includes edition suffixes for cleaning tests

### `lidarr-wanted-albums-sample.txt`
- **Format**: TXT (pipe-delimited)
- **Albums**: 10 classic albums
- **Purpose**: Human-readable format testing
- **Features**: Shows search query format for manual usage

### `lidarr-wanted-albums-sample.csv`
- **Format**: CSV
- **Albums**: 10 classic albums  
- **Purpose**: Spreadsheet import format testing
- **Features**: Proper CSV escaping for special characters

## Data Quality

All test data includes:
- ✅ Realistic artist and album names
- ✅ Proper search query formatting
- ✅ Album type diversity (album, single, ep)
- ✅ Year range from 1965-2023
- ✅ Track count variations
- ✅ Edition suffix cleaning examples
- ✅ Special characters in artist names (AC/DC, etc.)
- ✅ Various Artists compilation example
- ✅ Long album titles for edge case testing

## Usage in Tests

These files should be used to:
1. **Unit test export functionality** - Verify correct JSON/TXT/CSV formatting
2. **Test data parsing** - Ensure proper deserialization of export data  
3. **Validate search query generation** - Check clean title logic
4. **Test filtering and sorting** - Verify optimization algorithms
5. **Edge case handling** - Special characters, long titles, edition suffixes

## Updating Test Data

When the export functionality changes:
1. Regenerate sample files using the CLI export command
2. Verify all formats are properly structured
3. Update this README if new fields are added
4. Ensure backward compatibility with existing tests

## Export Command Examples

```bash
# Generate new JSON test data
qobuz lidarr export --format json --limit 25 --output lidarr-wanted-albums-sample.json

# Generate TXT format
qobuz lidarr export --format txt --limit 10 --output lidarr-wanted-albums-sample.txt

# Generate CSV format  
qobuz lidarr export --format csv --limit 10 --output lidarr-wanted-albums-sample.csv
```
