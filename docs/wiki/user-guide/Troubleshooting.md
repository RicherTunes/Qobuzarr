# Troubleshooting Guide\n\nThis comprehensive guide helps diagnose and resolve common issues with Qobuzarr. Follow the structured approach to quickly identify and fix problems.\n\n## Table of Contents\n\n- [Quick Diagnostics](#quick-diagnostics)\n- [Authentication Issues](#authentication-issues)\n- [Search Problems](#search-problems)\n- [Download Issues](#download-issues)\n- [Plugin Loading Errors](#plugin-loading-errors)\n- [Performance Issues](#performance-issues)\n- [API and Network Errors](#api-and-network-errors)\n- [Configuration Issues](#configuration-issues)\n- [Log Analysis](#log-analysis)\n- [Advanced Debugging](#advanced-debugging)\n- [Getting Help](#getting-help)\n\n## Quick Diagnostics\n\n### 🔍 Plugin Health Check\n\nStart with these quick checks to identify obvious issues:\n\n**1. Verify Plugin Status**\n```bash\n# Check if plugin is loaded\n# Navigate to: System → Status → Plugins\n# Look for \"Qobuzarr\" with \"Loaded\" status\n```\n\n**2. Test Connections**\n```bash\n# Test indexer: Settings → Indexers → Qobuzarr → Test\n# Test download client: Settings → Download Clients → Qobuzarr → Test\n# Both should return \"Test successful\"\n```\n\n**3. Quick Log Check**\n```bash\n# Linux/Docker\ntail -50 /config/logs/lidarr.txt | grep -i qobuz\n\n# Windows PowerShell\nGet-Content \"C:\\ProgramData\\Lidarr\\logs\\lidarr.txt\" -Tail 50 | Select-String \"qobuz\"\n\n# Look for obvious errors or authentication failures\n```\n\n### ✅ Health Indicators\n\n**Good Signs** (everything working):\n```\n[Info] QobuzarrModule: Plugin initialized successfully\n[Info] QobuzAuthenticationService: Authentication successful\n[Debug] QobuzIndexer: Search completed in 45ms\n```\n\n**Warning Signs** (needs attention):\n```\n[Warn] QobuzApiClient: Rate limited, waiting 60 seconds\n[Warn] QobuzAuthenticationService: Session expires in 1 hour\n```\n\n**Error Signs** (immediate action needed):\n```\n[Error] QobuzAuthenticationService: Authentication failed: Invalid credentials\n[Error] QobuzIndexer: Search failed: Network timeout\n[Fatal] PluginLoader: Failed to load Qobuzarr plugin\n```\n\n## Authentication Issues\n\n### 🔐 Invalid Credentials Error\n\n**Symptoms:**\n- \"Invalid credentials\" in test connection\n- 401 Unauthorized in logs\n- Search returns no results\n- \"Authentication failed\" errors\n\n**Step-by-Step Resolution:**\n\n**Step 1: Verify Credentials**\n```bash\n# Test with CLI first\ncd QobuzCLI\ndotnet run -- auth login --email \"your@email.com\" --password \"your_password\"\n\n# Expected success output:\n# Authentication successful!\n# Session token: abc123...\n# User: Your Name (your@email.com)\n```\n\n**Step 2: Check Account Status**\n- Log into <https://www.qobuz.com> directly\n- Verify account is active and subscription current\n- Check for any account restrictions or warnings\n- Try changing password and using new one\n\n**Step 3: Validate API Credentials**\n```bash\n# If using app credentials, test them\ncurl -X POST \"https://www.qobuz.com/api.json/0.2/user/login\" \\\n  -d \"app_id=YOUR_APP_ID\" \\\n  -d \"email=YOUR_EMAIL\" \\\n  -d \"password=YOUR_PASSWORD\"\n\n# Should return JSON with user_auth_token\n```\n\n**Step 4: Common Credential Issues**\n- **Special Characters**: Passwords with `&`, `%`, `+` may need URL encoding\n- **Email Format**: Ensure no typos, case sensitivity\n- **App Credentials**: Verify app_id and app_secret are correct\n- **Regional Issues**: Account region may not match content region\n\n### ⏰ Session Expired Errors\n\n**Symptoms:**\n- Works initially, fails after 24 hours\n- \"Session expired\" or \"Token expired\" in logs\n- Intermittent authentication failures\n\n**Solutions:**\n\n**Method 1: Force Session Refresh**\n```bash\n# Clear cached session\nrm /config/qobuz_session.cache  # Linux/Docker\ndel \"%APPDATA%\\Lidarr\\qobuz_session.cache\"  # Windows\n\n# Test connection again\n```\n\n**Method 2: Update Configuration**\n- Go to Settings → Indexers → Qobuzarr\n- Click \"Test\" button to force re-authentication\n- Save settings even if no changes made\n- Restart Lidarr if session issues persist\n\n**Method 3: Enable Auto-Refresh (if supported)**\n```json\n{\n  \"sessionManagement\": {\n    \"autoRefresh\": true,\n    \"refreshBuffer\": \"5min\",\n    \"tokenExpiry\": \"24h\"\n  }\n}\n```\n\n### 🔑 Token Authentication Issues\n\n**Symptoms:**\n- Token auth fails but email/password works\n- \"Invalid user ID or token\" errors\n- Token extracted from browser doesn't work\n\n**Resolution Steps:**\n\n**Step 1: Get Fresh Tokens**\n```javascript\n// In browser console on play.qobuz.com (after logging in)\nconsole.log('User ID:', localStorage.getItem('user.id'));\nconsole.log('Auth Token:', localStorage.getItem('user.userAuthToken'));\n```\n\n**Step 2: Validate Token Format**\n- **User ID**: Should be numeric (e.g., \"12345678\")\n- **Auth Token**: Alphanumeric string, usually long\n- **No Extra Characters**: Remove quotes, spaces, newlines\n\n**Step 3: Test Token Directly**\n```bash\n# Test with CLI using tokens\ndotnet run -- auth login --user-id \"12345678\" --auth-token \"your_token\"\n```\n\n## Search Problems\n\n### 🔍 No Search Results\n\n**Symptoms:**\n- Searches return empty results\n- \"No results found\" for albums known to exist on Qobuz\n- Plugin works but finds nothing\n\n**Diagnostic Steps:**\n\n**Step 1: Test Known Album**\n```bash\n# Test with a popular album that definitely exists\nqobuz search \"Pink Floyd Dark Side of the Moon\"\n\n# If this fails, it's likely an authentication or API issue\n# If this works, the problem is search-specific\n```\n\n**Step 2: Simplify Search Terms**\n```\n# Instead of:\n\"Pink Floyd - The Dark Side of the Moon (1973) [Remastered]\"\n\n# Try:\n\"Pink Floyd Dark Side Moon\"\n\n# Or even simpler:\n\"Dark Side Moon\"\n```\n\n**Step 3: Check Search Strategy**\n- Enable ML optimization (should be automatic)\n- Increase search result limit\n- Remove year/genre filters temporarily\n- Try different search approaches\n\n**Solutions:**\n\n**Method 1: Optimize Search Terms**\n```bash\n# Good search patterns:\nqobuz search \"Beatles Abbey Road\"\nqobuz search \"Miles Davis Kind Blue\"\nqobuz search \"Radiohead OK Computer\"\n\n# Avoid:\n- Special characters: - / \\ ( ) [ ]\n- Years or extra info: (1969), [Remastered]\n- Genre or format info: [FLAC], [Hi-Res]\n```\n\n**Method 2: Use ML-Optimized Search (Automatic)**\nThe plugin automatically optimizes searches using ML:\n- **~49% API call reduction** through smart query optimization\n- **Progressive search strategies** with fallbacks\n- **Context-aware simplification** based on query complexity\n\n### 🎯 Incomplete or Wrong Results\n\n**Symptoms:**\n- Missing known albums from search results\n- Only shows some formats/qualities\n- Results don't match Qobuz website\n\n**Solutions:**\n\n**1. Check Quality Filters**\n```json\n{\n  \"qualitySettings\": {\n    \"minimumQuality\": \"Any\",  // Don't restrict quality\n    \"enableAllFormats\": true,\n    \"includeMP3\": true,\n    \"includeFLAC\": true\n  }\n}\n```\n\n**2. Regional Availability**\n- Some content is region-locked\n- Your account region may not match content availability\n- Use VPN if needed (check Qobuz Terms of Service)\n\n**3. Subscription Limitations**\n- Hi-Res content requires Studio Premier subscription\n- Some exclusives may not be available via API\n- Check subscription tier supports requested quality\n\n### ⏱️ Search Timeouts\n\n**Symptoms:**\n- \"Request timeout\" errors\n- Searches take longer than 30 seconds\n- Intermittent failures under load\n\n**Performance Optimization:**\n\n**1. Reduce Search Load**\n```yaml\n# In Lidarr settings:\nSearch Result Limit: 50         # Reduce from 100\nInclude Singles: No             # Focus on albums\nInclude Compilations: No        # Reduce noise\nSearch Timeout: 60 seconds      # Increase if needed\n```\n\n**2. Enable ML Optimization**\nML optimization provides **~49% API call reduction**:\n```bash\n# Verify ML optimization is active\ngrep \"API call reduction\" /config/logs/lidarr.txt\n\n# Should see entries like:\n# [Info] CompiledMLQueryOptimizer: ~49% API call reduction achieved\n```\n\n**3. Check Network Performance**\n```bash\n# Test API response time\ntime curl -s \"https://www.qobuz.com/api.json/0.2/album/search?query=test&app_id=APP_ID\"\n\n# Should complete in <2 seconds\n```\n\n## Download Issues\n\n### 📥 Download Failures\n\n**Symptoms:**\n- Downloads start but fail to complete\n- \"Download failed\" in activity queue\n- Files partially downloaded or corrupted\n\n**Solutions:**\n\n**1. Check Download Permissions**\n```bash\n# Verify write permissions\ntouch /download/path/test.tmp && rm /download/path/test.tmp\n\n# Check disk space\ndf -h /download/path\n\n# Verify Lidarr user has access\nsudo -u lidarr touch /download/path/test.tmp\n```\n\n**2. Network and Connectivity**\n```bash\n# Test download URL accessibility\ncurl -I \"https://streaming.qobuz.com/test-url\"\n\n# Check for proxy/firewall issues\ntraceroute streaming.qobuz.com\n```\n\n**3. Quality and Subscription Issues**\n```json\n{\n  \"error\": \"Quality not available\",\n  \"message\": \"Requested quality requires Studio Premier subscription\",\n  \"available_qualities\": [5, 6],\n  \"requested_quality\": 27,\n  \"solution\": \"Lower quality setting or upgrade subscription\"\n}\n```\n\n### 🔄 Queue Management Issues\n\n**Symptoms:**\n- Downloads stuck in queue\n- Queue shows \"Downloading\" but no progress\n- Multiple failed download attempts\n\n**Solutions:**\n\n**1. Clear Failed Downloads**\n```bash\n# In Lidarr UI:\n# Activity → Queue → Clear failed items\n# Or manually remove stuck items\n```\n\n**2. Restart Download Service**\n```bash\n# Restart Lidarr to clear download state\ndocker restart lidarr  # Docker\nsudo systemctl restart lidarr  # SystemD\n```\n\n**3. Check Concurrent Download Limits**\n```json\n{\n  \"downloadSettings\": {\n    \"concurrentDownloads\": 3,  // Reduce if having issues\n    \"maxRetryAttempts\": 3,\n    \"retryDelay\": \"5s\"\n  }\n}\n```\n\n## Plugin Loading Errors\n\n### ❌ \"Could not load plugin\" Error\n\n**Symptoms:**\n- Plugin doesn't appear in Lidarr plugins list\n- \"Assembly loading failed\" in logs\n- \"Missing dependency\" errors\n\n**Step-by-Step Fix:**\n\n**Step 1: Verify Installation**\n```bash\n# Check plugin files exist\nls -la /config/plugins/Qobuzarr/\n\n# Required files:\n# - Lidarr.Plugin.Qobuzarr.dll (main assembly)\n# - plugin.json (plugin manifest)\n# - Dependencies (Newtonsoft.Json.dll, etc.)\n```\n\n**Step 2: Check .NET Runtime**\n```bash\n# Verify .NET 8.0 is available\ndotnet --version  # Should show 8.0.x or higher\n\n# In Docker containers:\ndocker exec lidarr dotnet --version\n```\n\n**Step 3: Validate Plugin Manifest**\n```json\n// Check plugin.json format\n{\n  \"name\": \"Qobuzarr\",\n  \"version\": \"0.0.12\",\n  \"lidarrVersion\": \"2.13.0.4686\",\n  \"assembly\": \"Lidarr.Plugin.Qobuzarr.dll\"\n}\n```\n\n**Step 4: Check File Permissions**\n```bash\n# Linux/Docker - fix permissions\nsudo chown -R lidarr:lidarr /config/plugins/\nsudo chmod -R 644 /config/plugins/Qobuzarr/*\nsudo chmod 755 /config/plugins/Qobuzarr/\n\n# Windows - run as administrator\nicacls \"C:\\ProgramData\\Lidarr\\plugins\" /grant lidarr:F /T\n```\n\n### 🔧 Plugin Version Incompatibility\n\n**Symptoms:**\n- Plugin loads but features missing\n- \"Minimum Lidarr version not met\" warning\n- Strange behavior or crashes\n\n**Solutions:**\n\n**1. Check Version Compatibility**\n```\nPlugin Version: 0.0.12\nRequired Lidarr: 2.13.0+\nYour Lidarr: Check in System → About\n```\n\n**2. Update Lidarr (if needed)**\n```bash\n# Docker - pull latest plugins branch\ndocker pull ghcr.io/hotio/lidarr:pr-plugins\ndocker-compose up -d lidarr\n\n# Manual - download from GitHub\nwget https://github.com/Lidarr/Lidarr/releases/latest\n```\n\n**3. Use Correct Plugin Branch**\nEnsure you're using plugins branch compatible assemblies:\n```bash\n# Plugin must be built against plugins branch\n# NOT regular release branch\ngit clone --branch plugins https://github.com/Lidarr/Lidarr.git\n```\n\n## Performance Issues\n\n### ⚡ Query Intelligence Not Working\n\n**Symptoms:**\n- No performance improvement after installation\n- API calls not reduced Performance:**\n```bash\n# Test with CLI to verify ML is working\ncd QobuzCLI\ndotnet run -- search \"Pink Floyd Dark Side Moon\" --debug\n\n# Should show:\n# Query complexity: Simple (Score: 0)\n# Optimized queries: 1 (reduced from 3, 66.7% reduction)\n# API calls saved: 2\n```\n\n### 🐌 Still Slow Despite ML Optimization\n\n**Symptoms:**\n- ML optimization working but searches still slow\n- API  // ~49% API call reduction\n    \"enableCaching\": true,               // 94.7% cache hit rate\n    \"cacheSize\": \"500MB\",\n    \"cacheTTL\": {\n      \"searchResults\": \"4h\",\n Monitor Memory Usage**\n```bash\n# Check memory usage\ndocker stats lidarr  # Docker\ntop -p $(pidof Lidarr)  # Linux\n\n# Baseline should be ~200MB\n# With caching ~500MB is normal\n# >1GB indicates memory leak\n```\n\n**2. Enable Adaptive Rate Limiting**\n```json\n{\n  \"rateLimiting\": {\n    \"enableAdaptive\": true,\n    \"maxRate\": 30,  // requests per minute\n    \"backoffMultiplier\": 2.0,\n    \"maxBackoff\": 300  // 5 minutes max\n  }\n}\n```\n\n**2. Reduce Request Load**\n```yaml\n# In Lidarr indexer settings:\nAPI Rate Limit: 30       # Reduce from 60\nSearch Result Limit: 50  # Reduce from 100\nDisable RSS: Yes         # Not needed for Qobuz\n```\n\n**3. Check for Request Loops**\n```bash\n# Look for repeated requests in logs\ngrep -c \"api.json\" /config/logs/lidarr.txt\n\n# High count (>1000) indicates possible loop\n```\n\n### 🌐 Network Connectivity Issues\n\n**Symptoms:**\n- Timeouts connecting to Qobuz API\n- \"Network unreachable\" errors\n- Works sometimes, fails other times\n\n**Diagnostic Steps:**\n\n**1. Test Basic Connectivity**\n```bash\n# Test DNS resolution\nnslookup www.qobuz.com\n\n# Test HTTPS connectivity\ncurl -I https://www.qobuz.com\n\n# Test API endpoint\ncurl -I https://www.qobuz.com/api.json/0.2/\n```\n\n**2. Check Proxy/Firewall**\n```bash\n# If using proxy, test with proxy settings\ncurl -x proxy:port https://www.qobuz.com\n\n# Check firewall rules\nsudo iptables -L | grep -i qobuz\n```\n\n**3. Docker Network Issues**\n```bash\n# Test from inside container\ndocker exec lidarr curl -I https://www.qobuz.com\n\n# Check container networking\ndocker network ls\ndocker inspect lidarr | grep -i network\n```\n\n### 🔥 Server Errors (500/502/503)\n\n**Symptoms:**\n- \"Internal server error\" from Qobuz\n- \"Bad gateway\" or \"Service unavailable\" errors\n- Intermittent failures\n\n**Solutions:**\n\n**1. Check Qobuz Status**\n- Visit <https://status.qobuz.com\n-> Check @QobuzOfficial Twitter for outages\n- Wait 5-10 minutes and retry\n\n**2. Implement Retry Logic** (automatic in plugin)\n```json\n{\n  \"retryPolicy\": {\n    \"maxAttempts\": 3,\n    \"backoffStrategy\": \"Exponential\",\n    \"baseDelay\": 1000,\n    \"maxDelay\": 30000\n  }\n}\n```\n\n**3. Report Persistent Issues**\nIf server errors persist for >1 hour:\n- Note exact error code and message\n- Check if specific to certain content\n- Report to plugin maintainers\n\n## Configuration Issues\n\n### ⚙️ Settings Not Saving\n\n**Symptoms:**\n- Configuration changes revert after restart\n- \"Settings saved\" but values don't persist\n- Default values keep returning\n\n**Solutions:**\n\n**1.
