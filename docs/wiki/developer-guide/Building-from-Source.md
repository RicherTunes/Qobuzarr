Manual Setup\n\nFor developers who prefer manual control or need to understand the process:\n\n### Step 1: Clone Repository\n\n```bash\n# Clone main repository\ngit clone https://github.com/RicherTunes/qobuzarr.git\ncd qobuzarr\n\n# Check available branches\ngit branch -r\n\n# Ensure you're on main branch\ngit checkout main\ngit pull origin main\n```\n\n### Step 2: Get Lidarr Dependencies\n\n**🚨 CRITICAL**: Use plugins branch assemblies for compatibility.\n\n**Option 1: Plugins Branch Source (Recommended)**\n```bash\n# Clone Lidarr plugins branch\ngit clone --depth 1 --branch plugins https://github.com/Lidarr/Lidarr.git ext/Lidarr-source\n\n# Verify correct branch\ncd ext/Lidarr-source\ngit branch\n# Should show: * plugins\n\ncd ../..\n```\n\n**Option 2: Docker Assembly Extraction**\n```bash\n# Create extraction directory\nmkdir -p ext/Lidarr-plugins/_output\n\n# Extract from plugins branch Docker image\ndocker pull ghcr.io/hotio/lidarr:pr-plugins-2.13.3.4692\ndocker create --name temp ghcr.io/hotio/lidarr:pr-plugins-2.13.3.4692\ndocker cp temp:/app/bin/. ext/Lidarr-plugins/_output/\ndocker rm temp\n```\n\n### Step 3: Apply Version Override\n\n**Critical Step**: Override Lidarr assembly version for compatibility.\n\n**PowerShell:**\n```powershell\n# Set target version (match your runtime Lidarr)\n$lidarrVersionOverride = \"2.13.2.4686\"\n\n# Apply version override\n(Get-Content \"ext\\Lidarr-source\\src\\Directory.Build.props\") -replace '<AssemblyVersion>[\\d\\.\\*]+</AssemblyVersion>', \"<AssemblyVersion>$lidarrVersionOverride</AssemblyVersion>\" | Set-Content \"ext\\Lidarr-source\\src\\Directory.Build.props\"\n\n# Verify override applied\nSelect-String -Path \"ext\\Lidarr-source\\src\\Directory.Build.props\" -Pattern \"AssemblyVersion\"\n```\n\n**Bash:**\n```bash\n# Set target version (match your runtime Lidarr)\nLIDARR_VERSION_OVERRIDE=\"2.13.2.4686\"\n\n# Apply version override\nsed -i \"s/<AssemblyVersion>[0-9.*]\\+<\\/AssemblyVersion>/<AssemblyVersion>$LIDARR_VERSION_OVERRIDE<\\/AssemblyVersion>/g\" ext/Lidarr-source/src/Directory.Build.props\n\n# Verify override applied\ngrep \"AssemblyVersion\" ext/Lidarr-source/src/Directory.Build.props\n```\n\n### Step 4: Restore Dependencies\n\n```bash\n# Restore NuGet packages\ndotnet restore\n\n# Verify packages restored\nls packages/  # Should contain restored packages\n\n# Check for restore errors\ndotnet restore --verbosity normal\n```\n\n### Step 5: Build Solution\n\n```bash\n# Build with required flags\ndotnet build --configuration Release \\\n  -p:RunAnalyzersDuringBuild=false \\\n  -p:EnableNETAnalyzers=false \\\n  -p:TreatWarningsAsErrors=false\n\n# Check build output\nls bin/Release/net6.0/\n# Should contain: Lidarr.Plugin.Qobuzarr.dll, plugin.json, dependencies\n```\n\n## Build Configuration\n\n### Project Structure Understanding\n\n```\nqobuzarr/\n├── Qobuzarr.csproj                 # Main plugin
